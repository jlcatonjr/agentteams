3Install Visual Studio Code

VS Code is free and available for Windows, macOS, and Linux from code.visualstudio.com/download.

The tab below is pre-selected based on a best-effort guess at your operating system — if it guessed wrong, just click the correct tab.

  1. Go to the download page and click Windows (the User Installer is recommended for most people).
  2. Run the downloaded .exe installer.
  3. Accept the license, keep the default install location, and on the "Select Additional Tasks" screen consider checking Add to PATH so the code command works in a terminal.
  4. Click Install, then Finish to launch VS Code.

4Install the GitHub Copilot extension

GitHub Copilot ships as a VS Code extension. There are two related extensions published by GitHub:

  • GitHub.copilot — inline code completions
  • GitHub.copilot-chat — the Chat panel, including its local "agent mode" for multi-step editor edits (see section 6 for how this differs from the GitHub-hosted coding agent)

Installing either one from the Extensions view will typically prompt you to install the other, since Copilot Chat depends on the completions extension. On some recent VS Code builds, Copilot Chat ships bundled by default — if a Chat icon already appears in the Activity Bar, you may not need to install it separately.

  1. Open VS Code.
  2. Open the Extensions view: click the square-icons icon in the Activity Bar, or press Ctrl+Shift+X (Cmd+Shift+X on macOS).
  3. Search for GitHub Copilot.
  4. Click Install on the "GitHub Copilot" extension published by GitHub.
  5. When prompted, also install GitHub Copilot Chat for the chat/agent panel.

If the code command is on your PATH, you can install from a terminal:

code --install-extension GitHub.copilot code --install-extension GitHub.copilot-chat
Requires a plan The extension installs for free, but using it requires a GitHub Copilot plan attached to your GitHub account (a free tier with monthly limits is available, alongside paid tiers such as Pro, Pro+, Business, and Enterprise). Tier names and limits change over time — verify the current lineup at github.com/features/copilot/plans. Organization-owned accounts may have Copilot access controlled by an administrator policy.

5Sign in and verify

  1. After installing, VS Code shows a Sign in to GitHub prompt — look for it as a notification, or open it yourself from the Accounts menu at the bottom of the Activity Bar (its exact position has moved across VS Code redesigns). Click it.
  2. Your browser opens to GitHub.com to authorize the "Visual Studio Code" OAuth app. Approve the request.
  3. Return to VS Code — it should show your GitHub username connected in the Accounts menu.
  4. Open any code file and start typing; gray "ghost text" suggestions from Copilot should appear inline.
  5. Open the Copilot Chat panel — click the chat icon in the Activity Bar, or open the Command Palette (Ctrl+Shift+P / Cmd+Shift+P) and run Chat: Open Chat — then send a test message such as "explain this file." (A direct keybinding also exists but has changed across VS Code releases, so the Command Palette route is the more durable instruction.)
No suggestions? Check the Copilot status icon in the VS Code status bar (bottom-right) — it reports sign-in state and whether completions are enabled for the current file type.

6The GitHub Copilot coding agent

Everything above sets up Copilot as an in-editor assistant — it completes code and answers questions while you drive. The Copilot coding agent is a further capability where Copilot works autonomously in the background: you (or a maintainer) assign it a GitHub issue, or ask it to make a change, and it works in its own environment, pushes commits to a branch, and opens a pull request for review — without you actively steering it turn-by-turn.

Two different "autonomous" features share the word "agent" The GitHub Copilot coding agent runs remotely on GitHub's infrastructure — not on your machine — and produces a pull request. You dispatch it either by assigning a GitHub issue to Copilot, or by starting a coding-agent session from Copilot Chat's session picker; both routes hand the work off to GitHub, not your local editor. VS Code Copilot Chat's local "agent mode," by contrast, runs directly in your editor against your open workspace and does not by itself open a pull request. Its availability depends on your Copilot plan and, for organizations, on policies enabled by an administrator. Exact plan requirements and rollout have changed over time — check docs.github.com/copilot for the current requirements before relying on this guide's description.

Typical flow for the remote coding agent, once available on your account/organization (the exact steps and UI have changed over time, so treat this as illustrative rather than exact):

  1. Open an issue on GitHub.com describing the change you want.
  2. Assign the issue to Copilot from the Assignees menu. If Copilot doesn't appear there, the feature likely isn't enabled for your account or organization yet.
  3. Copilot works in the background and opens a draft pull request, then updates it with a plan and commits.
  4. Review the PR like any other contribution — comment, request changes, or approve and merge.

The separate, local "agent mode" in VS Code's Copilot Chat panel is for editor-driven autonomous edits you can watch happen in real time on your machine — useful when you want to stay in the loop, rather than hand a task off to GitHub's remote agent entirely.
